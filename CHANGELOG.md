# Changelog — Changes from Upstream (`willchen96/mike`)

Per AGPL-3.0 §5(a), this file records that the Program has been modified, and when. This is the full per-file record; a summary appears in [`README.md`](./README.md).

### Turkish legal practice adaptation — 2026-07-02

The following changes adapt the product for Turkish law firms specifically (as opposed to the general/US-oriented legal assistant upstream provides):

- **Core assistant system prompt** (`backend/src/lib/chatTools.ts`) — rewritten to identify as a Turkish-law assistant: added core Turkish statute codes and abbreviations (TBK, TMK, TTK, HMK, İİK, TCK, CMK, İYUK, İş K., TKHK, VUK, Av.K.), the Turkish court hierarchy (first-instance courts, Bölge Adliye Mahkemesi / BAM, Yargıtay, Danıştay, Anayasa Mahkemesi), statute-of-limitations awareness, a rule against citing article/case numbers without tool verification, and explicit instructions to use the connected Yargı PRO MCP (Turkish legal-research MCP tool) (`mevzuat_ara`, `ictihat_ara`, `semantik_ictihat_ara`, `aym_ictihat_ara`) before US case-law tools.
- **Turkish court filing (dilekçe) generation** (`backend/src/lib/chatTools.ts`, `generate_docx` tool) — added a `layout: "petition"` mode that renders section headings unnumbered (matching Turkish court-filing convention, unlike contract clause numbering), right-aligns the closing tarih/imza block, and restarts numbering per section so demands under "SONUÇ VE İSTEM" don't continue counting from the fact narrative. Fixed a bug where the manual-list-marker parser stripped abbreviations like "Av." from signature lines.
- **Seven Turkish civil-procedure petition workflows** added (`backend/src/lib/builtinWorkflows.ts` and `frontend/src/app/components/workflows/builtinWorkflows.ts`, kept in sync): dava dilekçesi, cevap dilekçesi, cevaba cevap (replik), ikinci cevap (düplik), istinaf başvuru dilekçesi, temyiz başvuru dilekçesi, and bilirkişi raporuna itiraz dilekçesi — each grounded in the relevant HMK articles.
- **"Bilirkişi Raporu İncelemesi" tabular review template** added (`frontend/src/app/components/workflows/builtinWorkflows.ts`) — 14 columns for reviewing Turkish court-appointed expert reports (bilirkişi raporu), including a column flagging when an expert has exceeded their mandate under HMK m. 266 by making legal (rather than technical) determinations.
- **Existing tabular review templates bound to Turkish law** (`frontend/src/app/components/workflows/builtinWorkflows.ts`) — the share purchase agreement, credit agreement, supply agreement, employment agreement, e-discovery, and venture/LP fund review templates were generic common-law-style prompts; their columns now cite the relevant TBK, TTK, İş Kanunu (4857), and Avukatlık Kanunu provisions (e.g. TBK m. 115 on the non-waivability of gross-negligence liability, İş K. m. 17/25/26/53 on notice periods and leave entitlements, Av.K. m. 36 in place of a common-law "privilege" framing). A new "Rekabet Etmeme" column (TBK m. 444–447) was added to the employment template; two new columns (CFC/tax reporting under Kurumlar Vergisi Kanunu m. 7, and TPKKM capital-movement notification) were added to the fund review template, since Turkey has no direct equivalent to a foreign limited-partnership vehicle.
- **Practice-area list** (`frontend/src/app/components/workflows/practices.ts`) — replaced the Anglo-American abbreviations "EC/VC", "ECM", "DCM" with "Girişim Sermayesi" and a merged "Sermaye Piyasası Hukuku", matching how Turkish firms actually group these practice areas.
- **US case-law research (CourtListener) is now opt-in, off by default** — previously every new user had this enabled by default. Changed the `legal_research_us` column default in `backend/schema.sql` and a new migration (`backend/migrations/20260701_user_profile_legal_research_us_default_off.sql`), plus the corresponding fallback logic in `backend/src/lib/userSettings.ts`, `backend/src/routes/user.ts`, and `frontend/src/contexts/UserProfileContext.tsx` / `frontend/src/app/(pages)/account/features/page.tsx`. `frontend/src/app/components/assistant/ChatView.tsx` was also changed so the CourtListener case-law panel cannot be opened while the feature is disabled for the user, even from historical citations.
- **Tabular review creation UX** (`frontend/src/app/components/tabular/AddNewTRModal.tsx`) — added inline hints explaining why the "Oluştur" (Create) button is disabled (missing title / missing selected project), since neither previously had any visible validation message.
- **Credits-exhausted modal** (`frontend/src/app/components/modals/credits-exhausted-modal.tsx`) — this component had been missed during Turkish localization; translated it in full and fixed its date formatting from `en-US` to `tr-TR`.

### Earlier changes (product rebrand, localization, and Turkish-market adaptation)

The rest of this fork's ~90 remaining changed files predate the Turkish-legal-practice work above. Summarized by theme:

- **Branding (Mike → RapidAct).** Product name changed throughout the app (page titles/metadata, sidebar logo, server startup logs, the default DOCX tracked-change author, the MCP OAuth client name, and the assistant's self-identification). The favicon and chat status icon were simplified from a multi-gradient logo to a single-path chevron mark. Exported file names and the MFA authenticator entry name were updated to match.
- **Turkish localization (i18n).** Nearly all frontend components (account, assistant, projects, tabular reviews, workflows, shared components — roughly 90 files) were translated from English to Turkish: button labels, form fields, error/confirmation messages, tooltips, empty-state copy. Root `layout.tsx` sets `<html lang="tr">`, page metadata was translated, and the `latin-ext` Google Fonts subset was added for Turkish characters (ç, ğ, ı, ö, ş, ü). Date/time formatting (`toLocaleDateString`, `Intl.DateTimeFormat`) was pinned to the `"tr-TR"` locale. Backend-side LLM system prompts (chat title generation, project chat, tabular review extraction/chat) were rewritten to produce Turkish output; default document/review names ("Untitled document" → "Adsız belge") were localized. `AssistantMessage.tsx` uses the Turkish legal citation convention "a.g.e." instead of "Id."; `CaseLawPanel.tsx` opinion-type terminology was localized (e.g. "Lead Opinion" → "Asıl Görüş").
- **Turkish-market field adaptation.** `columnPresets.ts` tabular-review column presets (Parties, Governing Law, Effective Date, etc.) were extended to match Turkish contract/document terminology via regex. `pillUtils.ts` added cell-coloring support for TRY/TL currency codes and Turkish yes/no values. `RowActions.tsx` renamed the "CM No." (Client Matter Number) field to "Dosya No" to match Turkish law-firm terminology. Tabular review currency examples were updated from `$1,234.56`/USD to `1.234,56 TL`/TRY.
- **Bug fixes.** `userApiKeys.ts` — fixed key resolution for providers with multiple stored key rows (now uses the latest valid key instead of the first found). `mikeApi.ts` — added Unicode NFC normalization to all string values in API responses, fixing display/comparison inconsistencies from Turkish character encoding variance. `api-keys/page.tsx` — removed a restriction that prevented users from overriding a server-`.env`-configured API key from the UI.
- **UI/UX.** `HeaderActionsMenu.tsx` widened the actions menu to fit longer Turkish labels. `ConfirmPopup.tsx` recognizes the Turkish "sil" (delete) label for its destructive-action styling. `NewWorkflowModal.tsx` renamed the "Others" practice-area option to "Diğer".
